Core Web Vitals คืออะไร ทำไมถึงสำคัญต่อ SEO ของคุณ

Google ประกาศใช้ Core Web Vitals เป็นหนึ่งในปัจจัยการจัดอันดับตั้งแต่ปี 2021 แต่หลายเจ้าของเว็บไซต์ยังไม่รู้ว่ามันคืออะไร ทำงานยังไง และต้องทำอะไรเพื่อให้ผ่านเกณฑ์ บทความนี้จะอธิบายครบทุกประเด็น

Core Web Vitals คืออะไร?

Core Web Vitals คือชุดตัวชี้วัดที่ Google กำหนดขึ้นเพื่อวัด "ประสบการณ์ผู้ใช้งาน" ของเว็บไซต์ในแง่ที่จับต้องได้ ปัจจุบันมี 3 ตัวชี้วัดหลัก ได้แก่ LCP, INP และ CLS ซึ่งแต่ละตัววัดคนละมิติของ UX

ทำไมต้องสนใจ? เพราะ Google รวม Core Web Vitals เข้ากับ Page Experience signal และนำมาใช้เป็นปัจจัยหนึ่งในการจัดอันดับผลการค้นหา เว็บที่ผ่านเกณฑ์มีโอกาสติดอันดับสูงกว่าเว็บที่ไม่ผ่าน เมื่อ content คุณภาพเทียบเท่ากัน

LCP — Largest Contentful Paint

LCP วัดเวลาที่ใช้ในการแสดง element ที่ใหญ่ที่สุดที่มองเห็นได้ในหน้าจอ ซึ่งมักเป็นรูปภาพ hero หรือ heading ขนาดใหญ่ เป็นตัวแทนของความรู้สึก "เว็บโหลดเสร็จแล้ว" ของผู้ใช้งาน

ระดับ เวลา LCP
ดี (Good) ≤ 2.5 วินาที
ต้องปรับปรุง 2.5 – 4.0 วินาที
แย่ (Poor) > 4.0 วินาที

สาเหตุหลักที่ทำให้ LCP แย่:

  • รูปภาพขนาดใหญ่ที่ไม่ได้ compress หรือใช้ format ที่เหมาะสม (ควรใช้ WebP)
  • Server response time ช้า (TTFB สูง)
  • Render-blocking resources เช่น CSS หรือ JavaScript ขนาดใหญ่
  • ไม่ได้ preload รูปภาพ hero ที่สำคัญ

INP — Interaction to Next Paint

INP แทนที่ FID (First Input Delay) ตั้งแต่มีนาคม 2024 โดยวัดความเร็วในการตอบสนองต่อการโต้ตอบทั้งหมดของผู้ใช้งาน เช่น การคลิกปุ่ม การพิมพ์ในช่องข้อความ หรือการแตะบนมือถือ

ระดับ เวลา INP
ดี (Good) ≤ 200 ms
ต้องปรับปรุง 200 – 500 ms
แย่ (Poor) > 500 ms

INP แย่มักเกิดจาก JavaScript ที่หนักเกินไปที่ block main thread ทำให้ browser ไม่สามารถตอบสนองต่อ input ของผู้ใช้งานได้ทันที

CLS — Cumulative Layout Shift

CLS วัด "การเลื่อนหน้าเว็บแบบไม่คาดคิด" ที่เกิดระหว่างการโหลด เช่น ปุ่มหรือข้อความที่ขยับตำแหน่งกะทันหันหลังรูปภาพ โหลดเสร็จ ทำให้ผู้ใช้งานคลิกผิดหรือสับสน

ระดับ คะแนน CLS
ดี (Good) ≤ 0.1
ต้องปรับปรุง 0.1 – 0.25
แย่ (Poor) > 0.25

วิธีแก้ CLS:

  • กำหนด width และ height attribute ของรูปภาพและ video ทุกชิ้นเสมอ
  • ไม่ใส่ content ใหม่ด้านบนของ content เดิมแบบ dynamic
  • ใช้ font-display: swap เพื่อป้องกัน layout shift จาก web font
  • Reserve space สำหรับ ads หรือ embed ที่โหลดทีหลัง

วิธีตรวจสอบ Core Web Vitals

เครื่องมือที่แนะนำ:

  • Google PageSpeed Insights — วิเคราะห์ URL ใดก็ได้ฟรี พร้อมคำแนะนำแก้ไขเฉพาะจุด
  • Google Search Console → Core Web Vitals report — แสดงภาพรวมของเว็บทั้งหมดตาม field data จริง
  • Chrome DevTools Lighthouse — ทดสอบบนเครื่องของคุณก่อน deploy
  • web.dev/measure — วิเคราะห์อย่างละเอียดพร้อม learning resources

Static Website กับ Core Web Vitals

เว็บไซต์ static ที่ host บน CDN มีข้อได้เปรียบตามธรรมชาติ เพราะ TTFB ต่ำมาก ไม่มี server-side processing ทำให้ LCP ดีกว่า CMS โดยทั่วไป อย่างไรก็ตาม ยังต้องระวังเรื่องรูปภาพขนาดใหญ่และ JavaScript ที่หนัก

เว็บไซต์ที่ WebExpressTH สร้างผ่าน Google PageSpeed Insights ในระดับ 90+ คะแนน สำหรับทั้ง Mobile และ Desktop เป็นมาตรฐานที่เราตั้งไว้ตั้งแต่วันแรก

สรุป: Core Web Vitals เป็นตัวชี้วัดที่จับต้องได้ ไม่ใช่แค่ตัวเลขสวยหรู แต่สะท้อนประสบการณ์จริงของผู้เข้าชมเว็บ เว็บที่โหลดเร็ว ตอบสนองไว และไม่กระตุก ย่อมสร้าง impression ที่ดีกว่าและ convert ได้ดีกว่าในระยะยาว

เขียนโดย WebExpressTH — เราสร้างเว็บไซต์ที่ผ่านเกณฑ์ Core Web Vitals ตั้งแต่วันแรก

คำถามที่พบบ่อย

Core Web Vitals สำคัญกว่า content ไหม?

ไม่ครับ Content คุณภาพยังเป็นปัจจัยหลัก Core Web Vitals เป็นปัจจัยเสริมที่ Google ใช้ตัดสินเมื่อ content คุณภาพใกล้เคียงกัน แต่เว็บที่โหลดช้ามากอาจส่งผลเสียต่อ user experience จนทำให้ bounce rate สูงได้

WordPress ผ่าน Core Web Vitals ได้ไหม?

ได้ครับ แต่ต้องลงทุนเพิ่มในการ optimize — ใช้ caching plugin ที่ดี, image optimization, CDN และ theme ที่เบา อย่างไรก็ตาม static site มีข้อได้เปรียบตามโครงสร้างและทำได้ง่ายกว่ามาก

ตรวจสอบ Core Web Vitals ฟรีได้ไหม?

ได้ครับ ทั้ง Google PageSpeed Insights, Search Console และ Chrome DevTools เป็นเครื่องมือฟรีทั้งหมด ไม่ต้องเสียเงินซื้อ tool อะไรเพิ่ม
